# 35_资管公司_智能客服与自动回访_需求拆解与页面方案

## 1. 项目目标与范围（MVP）

### 1.1 目标
围绕“智能客服与自动回访”建设可演示的管理后台，支持：
- 在线咨询问答（文本机器人）
- 自动回访任务管理与执行记录
- 看房预约管理
- FAQ 知识库维护与命中反馈

### 1.2 MVP 范围
- 仅实现前端可感知闭环，使用 Prisma + SQLite 本地演示数据。
- 不接入真实语音外呼、短信、第三方客服系统。
- AI 能力优先使用本地 Mock 结果，保留后续接入真实模型接口的扩展位。

### 1.3 非目标（本期不做）
- 真实呼叫中心 SIP/语音网关对接
- 真实客户身份核验与计费
- 复杂排班引擎与自动质检

---

## 2. 业务对象与数据模型（MVP）

统一沿用示例项目 `InvestModel` 作为通用工作项承载，按业务语义映射为“客服/回访任务单”。

核心字段语义映射：
- `code`: 任务编号（如 CSR-2026-001）
- `title`: 咨询主题/回访主题
- `modelType`: 任务类型（在线咨询/回访任务/看房预约/知识优化）
- `industry`: 客服渠道（公众号/小程序/电话/企业微信）
- `region`: 项目区域或小区
- `status`: 状态流转（待接待 -> 处理中 -> 已回访 -> 已完成）
- `riskLevel`: 客诉风险等级（高/中/低）
- `owner`/`analyst`: 客服坐席/回访专员
- `inputParams`: 客户诉求与背景
- `outputResult`: 处理结果摘要
- `prediction`: 回访建议/满意度预测/下次动作建议

## 3. 页面信息架构

## 3.1 主业务页面
1. 客服首页（dashboard）
- 今日咨询量、待回访数、高风险客诉、闭环率
- 渠道健康度与服务洞察
- 最近服务留痕（时间线）

2. 在线咨询（consultation）
- 咨询工单列表
- 新建咨询记录
- 智能问答（Ant Design X）
- 转人工与状态推进

3. 回访管理（callback）
- 回访任务列表
- 回访任务新建
- 自动回访结果登记
- 删除前二次确认

4. 知识库（knowledge）
- FAQ 条目列表
- FAQ 新增与维护
- 命中效果统计（Mock）

5. 智能客服助手（assistant）
- 会话历史
- 快捷提问（解答、回访话术、预约提醒）
- 一键采纳结果写入业务留痕

## 3.2 管理页面（与示例保持一致）
- 用户管理（users）
- 日志审计（audit-logs）
- 系统设置（settings）

---

## 4. 关键交互闭环

### 4.1 在线咨询闭环
- 新建咨询 -> 列表新增 -> AI 生成建议 -> 转人工/推进状态 -> 留痕更新

### 4.2 回访闭环
- 创建回访任务 -> 指派专员 -> 记录回访结果 -> 客户确认预约 -> 标记完成

### 4.3 知识库闭环
- 新增 FAQ -> 在智能问答中被引用 -> 命中统计变化（Mock）

### 4.4 高风险确认机制
- 对 `riskLevel=高` 的任务，推进和删除都需二次确认。

---

## 5. 页面布局与组件拆分方案

### 5.1 布局层
- `ConsoleShell` 复用示例项目：左侧导航 + 顶部面包屑 + 内容区。
- 保持蓝白浅色 B 端风格，适配桌面端。

### 5.2 业务组件层
- `DashboardClient`：按路由渲染首页、工作台、分析页、管理页。
- `AgentWorkbench`：承载 AI 会话、快捷提问、采纳写回。
- `ModelComparisonRadar`：复用图表组件，语义改为服务质量/风险对比。

### 5.3 状态管理层
- 页面级：`useState` + `useTransition`
- 跨页面：服务端 `getDashboardSnapshot` 汇总 + Server Actions 写回

---

## 6. Mock 数据策略

- 咨询语料使用真实业务风格：租金缴纳、合同续签、看房预约、维修投诉。
- 回访记录包含客户姓名、电话脱敏、回访时间、结果、满意度。
- FAQ 包含常见高频问题并关联推荐回复。
- 统计卡片与图表均基于 SQLite 中种子数据计算，不使用 test1/test2 占位。

---

## 7. 与真实服务对接改造点（后续）

1. 语音机器人
- 接入外呼平台回调，替换 `modelEvent` 中的 Mock 回访结果。

2. 在线客服
- 对接 IM 会话流，替换当前本地会话存储。

3. 知识库
- 引入向量检索/检索增强，替换 FAQ 规则匹配。

4. 预约系统
- 对接真实房源与日历服务，替换本地预约字段。

---

## 8. 验收标准（MVP）

- 登录后可进入统一控制台。
- 四个业务页面可正常访问，布局一致。
- 每个页面可完成至少一条“新增 -> 状态变化 -> 留痕更新”闭环。
- AI 助手可生成回复并采纳，页面刷新后可保留历史会话。
- 用户/日志/设置三类管理页可用。
- `npm run typecheck` 与 `npm run build` 通过。
